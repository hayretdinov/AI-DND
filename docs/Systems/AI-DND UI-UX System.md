# AI-DND UI-UX System

AI-DND

           UI_UX_SYSTEM.md

           User Interface and User Experience System

           Version 0.1

           1. DESIGN    PHILOSOPHY

           The interface must disappear.

           The player should feel:

           "I am inside the world."

           Not:

           "I am using software."

           2. VISUAL   STYLE

           Primary inspirations:

              • Gothic 1
              • Morrowind
              • Darkest Dungeon
              • Medieval manuscripts

           Style:

              • dark fantasy;
              • old parchment;
              • worn wood;
              • iron;
              • candle light.

3. CORE   PRINCIPLES

           The interface must be:

              • immersive;
              • minimal;
              • readable;
              • atmospheric.

           4. WHAT   WE   DO  NOT   USE

           Never use:

              • floating damage numbers;
              • quest markers;
              • exclamation marks;
              • minimaps;
              • glowing paths;
              • objective arrows;
              • skill trees UI.

           5. MAIN   GAME    SCREEN

           Screen consists of:

              • world scene;
              • thought panel;
              • dialogue panel;
              • status panel;
              • quick actions.

           6. THOUGHT      PANEL

           Purpose:

           Display character thoughts.

           Examples:

                                          2

<!-- Page 3 -->

• This road seems dangerous. • I should rest. • Something is wrong.

           Thoughts are not commands.

           7. DIALOGUE     PANEL

           Contains:

              • NPC speech;
              • player input;
              • narration.
           Player types freely.

           8. PLAYER   INPUT

           Examples:

           "I attack."

           "I negotiate."

           "I investigate."

           No buttons required.

           9. SUGGESTION      SYSTEM

           Suggestions appear as:

           character thoughts.

           Examples:

           • Perhaps negotiation is possible. • He seems frightened.

10. COMBAT     SCREEN

           Perspective:

           First-person psychological view.

           The player sees:

              • enemy;
              • environment;
              • effects.

           The player does not see:

           their own body.

           11. TARGETING

           Initially:

           only torso.

           Later:

              • arms;
              • legs;
              • head;

              • special targets.
           Unlocked through training.

           12. HEALTH    DISPLAY

           No health bars.

           Instead:

              • wounds;
              • blood;
              • pain;
              • thoughts.

13. ENEMY    CONDITION

           Examples:

              • breathing heavily;
              • limping;
              • bleeding;
              • frightened.

           14. PLAYER    CONDITION

           Examples:

           • My arm hurts. • I feel weak. • I am exhausted.

           15. DICE  DISPLAY

           Dice remain visible.

           Examples:

           D20: 17

           Attack: Success.

           16. INVENTORY

           Visual style:

              • backpack;
              • pockets;
              • equipment layout.

           Items appear physical.

17. MAP   SCREEN

           Visual style:

              • old parchment;
              • hand-drawn maps;
              • notes.

           18. JOURNAL

           Journal stores:

              • observations;
              • memories;
              • rumors.

           Not objectives.

           19. CHARACTER      SCREEN

           Displays:

              • biography;
              • skills;
              • injuries;
              • fears;
              • reputation.

           20. SKILL  DISPLAY

           Display:

           names.

           Hide:

           internal calculations.

21. MAGIC    INTERFACE

           Displays:

              • knowledge;
              • rituals;
              • spells.

           Not cooldown bars.

           22. THOUGHT      SETTINGS

           Modes:

              • Full Thoughts;
              • Intuition Only;

              • Combat Only;
              • Hardcore.

           23. DIALOGUE      HISTORY

           Stores:

              • conversations;
              • promises;
              • threats.

           24. RUMOR     INTERFACE

           Rumors appear naturally.

           Example:

           "I heard something happened."

           25. QUEST    INTERFACE

           Display:

                                          7

stories.

           Do not display:

           tasks.

           26. WORLD     NEWS

           Examples:

              • wars;
              • plagues;
              • disasters.

           Presented through:

              • rumors;
              • newspapers;
              • NPCs.

           27. STONE    SLEEP  SCREEN

           When awakening:

           show world changes.

           Example:

           "You slept for 12 days."

           28. ACCESSIBILITY

           Optional:

              • subtitles;

              • text scaling;
              • assistance.

29. IMMERSION       RULE

           UI elements should appear as objects from the world.

           30. CORE   RULE

           The player should remember the world.

           Not the interface.
